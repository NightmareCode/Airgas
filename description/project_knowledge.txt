AIRGAS TECHNOLOGY - PROJECT KNOWLEDGE BASE

1. PROJECT OVERVIEW
------------------
The Airgas Technology Landing Page is a modern, high-performance single-page application (SPA) designed as a product catalog for industrial safety equipment and gas solutions in Malaysia. It uses a "Vanilla" tech stack (HTML/CSS/JS) for maximum speed and portability.

2. CORE FILE ARCHITECTURE
-----------------------
- index.html: The structural foundation. Contains sections for navigation, the product catalog UI, contact information, and the footer.
- style.css: A custom "Modern Industrial" design system. Features include:
    * CSS variables for theme consistency.
    * Responsive grid layouts (adaptive from desktop to small mobile).
    * Sophisticated animations (staggered reveals, backdrop blurs, glassmorphism).
- script.js: The application logic. Responsible for:
    * Fetching and parsing 'assets/products.json'.
    * Dynamic rendering of product cards.
    * Real-time search and category filtering.
    * Interactive 3D hover tracking on images.
    * Mobile navigation toggle and scroll-based UI updates.
- review.html: A developer tool for visual audits. It maps products to their images to check for naming errors or missing assets.

3. DATA & ASSETS
---------------
- assets/products.json: The central data store. Contains arrays for categories, contact details (including WhatsApp numbers), and the full product list (id, name, category, brand, brandUrl).
- assets/ (Category Folders): Images are organized into folders based on category names (e.g., 'PPEForERT', 'PPEAtWorks', 'Gases').
- assets/BrandLogo/: Brand-specific logos used in the product cards.

4. NAMING CONVENTIONS & WORKFLOWS
-------------------------------
- Product Images: Generated dynamically in JS.
    * Logic: product_name -> remove non-alphanumeric -> trim -> .png
    * Example: "MSA M1 SCBA" becomes "MSA M1 SCBA.png"
- Brand Logos: lowercased_brand_name.png (with spaces to underscores).
- Fallbacks: If an image fails to load, the UI automatically falls back to text-based initials.

5. MAINTENANCE TOOLS (tools/ directory)
--------------------------------------
The project includes several Python-based automation scripts:
- fetch_product_images.py: Automates the sourcing of missing product images.
- image_trimmer.py: Removes transparent margins for cleaner UI display.
- image_enhancer.py: Upscales or improves image quality.
- fix_bad_images.py & fix_escape_hood.py: Targeted scripts for specific asset corrections.
- bulk_download.py & image_downloader.py: Core utilities for handling external image requests.

6. UI/UX FEATURES
----------------
- Staggered Reveal: Cards animate in sequence for a premium feel.
- 3D Hover: Images follow the mouse cursor with subtle translation and scaling.
- Search/Filter: Instant UI updates based on user input without page reloads.
- Mobile Responsive: Custom breakpoints for tablets and small phones (grid switches to 2-columns).

8. COMPANY BACKGROUND & SPECIALIZATION

CORE SPECIALIZATIONS:
